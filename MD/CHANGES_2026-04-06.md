# Todo App Change Report - 2026-04-06

## Scope of today's work

This report covers the functional work completed today across the frontend and backend:

- Added a database relationship between tasks and board columns.
- Added backend support for column-aware task reads/writes.
- Added an endpoint to rename column labels and persist those labels.
- Wired frontend API and state logic to persist column renames.
- Added fallback behavior so empty column names resolve to defaults.
- Updated project documentation placement and navigation files.

## Codebase architecture refresher

The app uses a layered architecture:

- Frontend: React + hooks + service layer for HTTP and realtime subscription.
- Backend: Fastify route layer + Zod validation + service layer + repository layer.
- Database: MSSQL through Knex migrations and query builder.

Request path for task operations:

1. UI interaction in board/hooks.
2. HTTP call in frontend service.
3. Fastify route validation.
4. Business logic in service.
5. SQL persistence in repository.
6. Response returned to frontend.

Realtime path:

1. Backend publishes events after mutations.
2. Frontend listens through EventSource.
3. Local state updates from incoming snapshot/events.
